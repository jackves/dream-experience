Core Concept

立意：AI 作为可交互的外部梦境

梦是人脑的生成式模拟——它不评判、不给建议，只是把情绪翻译成象征语言。

如果 AI 能做同样的事呢？不是替你想清楚问题，而是像梦一样，把那些说不出口的重量，变成你可以触碰的样子。

★

核心立意

「梦是人脑的生成式模拟，AI 是可交互的外部梦境。」用户输入现实困扰，系统不直接给建议，而是把困扰转译成一场可探索的梦。

☷

面向人群

有压抑、疲惫、迷茫、焦虑、说不出口情绪的人。体验应该像进入一场真实的梦，而不是看一个产品说明。

◆

设计原则

游戏内不出现任何跳戏内容（比赛、产品化、AI 说明等）。故事完整、有连续性，每层选择影响下一层。

☆

情感目标

玩家在梦中触碰物件 → 物件产生回应 → 带着感受自然醒来。整个过程没有说教，只有陪伴和转译。

「这不是心理咨询，也不是什么 AI 在替你想清楚一切。
只是有时候，我们需要一个地方，
把那些说不出口的东西，换一种方式看见。

然后发现——原来它们没有想象中那么重。」
—— 来自终章收尾文字

Implementation

技术实现

纯前端方案，无需后端部署。使用 React 组件化架构 + Framer Motion 动画引擎 + Zustand 状态管理，实现沉浸式多场景切换与 AI 梦译机制。

React 18 TypeScript Vite 4 Tailwind CSS Framer Motion Zustand (状态管理) 纯 SVG 手绘风格物件 CSS Keyframes 动画 AI 关键词梦译引擎 5 条完整故事线

AI Dream Translation

AI 梦译机制

用户输入原始文本后，系统通过关键词匹配进行情绪分类，然后映射到对应的物件链、场景文案和终章感悟。整个 AI 机制隐藏在后端，用户只感受到「梦懂了我」。

用户情绪 示例输入 梦译输出 第一层物件 第二层物件 终章感悟
疲惫 "我很累，快撑不住了" 「背包太重了」 旧伞 沉重的影子 你不必一直撑着
焦虑 "怕做不好，怕让人失望" 「前面看不清路」 路灯 雾窗 光会亮起来的
迷茫 "不知道该怎么办" 「找不到要坐哪班车」 空白车票 时刻表 车会来的
孤独 "有些话说不出口" 「信写好了但没有地址」 旧信封 空座位 有人会读到的
关系困扰 "和某人之间出了问题" 「两个人隔着玻璃」 碎镜 窗中倒影 玻璃可以擦亮的
